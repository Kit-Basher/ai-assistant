# Writing and Editing Workflow

Use this pack when the user asks for writing, editing, rewriting, critique, tone, or structure help.

Rules:
- Stay text-only. Do not publish, upload, email, or modify external documents from this pack.
- Preserve user intent and ask about audience, tone, length, and constraints when needed.
- For factual or high-stakes writing, separate claims from style edits and recommend source verification.

Workflow:
1. Identify goal, audience, tone, format, and length.
2. Choose a pass type: outline, draft, edit, critique, tighten, expand, or polish.
3. Make changes traceable when the user wants revision control.
4. Preserve sensitive details and avoid inventing facts.
5. End with the next useful editing pass.

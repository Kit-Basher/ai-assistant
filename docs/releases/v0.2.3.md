# Personal Agent v0.2.3 Release Notes

Status: patch candidate.

v0.2.3 is a focused product-quality patch on top of the verified v0.2.2
authorization and reproducibility foundation.

## Fixes

- Added Telegram transport health diagnostics that distinguish configured,
  polling-active, handler-registered, inbound-update, outbound-reply, duplicate
  consumer, and degraded states.
- Added fixture Telegram transport proof for update -> handler -> runtime ->
  fake reply delivery.
- Improved local-device intent routing so explicit “my PC”, “this machine”,
  RAM, CPU, GPU, disk, and process questions use trusted local read-only
  inspection instead of web search.
- Added bounded process-level memory and CPU inspection.
- Added grouped process names and friendly application labels.
- Added memory-focused system-check answers that lead with RAM pressure and top
  memory users.
- Distinguished system RAM from GPU VRAM in user-facing responses.

## Security

- Local inspection remains read-only.
- Process inspection omits command lines and environment variables by default.
- Telegram diagnostics never expose bot tokens or raw message bodies.
- Telegram repair actions remain bounded and authorization-gated when invoked.
- The v0.2.2 capability policy, Universal Plan, executor registry, and mutation
  bypass hardening remain in force.

## Known limitations

- Live Telegram delivery still depends on Telegram availability and valid local
  configuration.
- The default Telegram diagnostic is no-send; a live reply canary requires
  explicit user intent.
- Per-process GPU memory depends on driver support and may not be available.
- Process labels are best-effort and may fall back to generic executable names.
- Arbitrary malicious in-process Python is still not process-isolated.
- Primary uninstall remains disabled without local activation, and purge remains
  unsupported.

# Personal Agent v0.2.4 Release Notes

v0.2.4 is a focused Telegram reliability and secrecy patch on top of v0.2.3.

## Fixes

- Redacts Telegram bot-token URLs from logs, diagnostics, audit-style events,
  and support evidence paths.
- Adds regression coverage for Telegram API URL, exception, and nested payload
  redaction using fake tokens only.
- Adds redacted Telegram live reply tracing for received, dispatched, attempted,
  succeeded, and failed reply states.
- Distinguishes configured, polling, receiving, dispatching, replying, healthy,
  degraded, and unverified Telegram states.
- Handles `are you ther?`, `hello?>`, `ping`, and similar greetings through a
  fast local response path.
- Improves fallback wording for timeout, local backend, and malformed response
  cases.

## Security

- Telegram tokens, authorization headers, raw update payloads, message bodies,
  and full chat ids are not logged by default.
- Local inspection and Telegram diagnostics remain read-only.
- Authorization and mutation boundaries from v0.2.2/v0.2.3 are unchanged.

## Operator Action

If any previous log contained `https://api.telegram.org/bot...`, rotate the bot
token in BotFather, update the Personal Agent secret store, and restart
`personal-agent-telegram.service`.

## Known limitations

- This patch does not rotate secrets automatically.
- Live Telegram delivery depends on the optional Telegram service being active.
- Live Telegram canary remains explicit and optional because it involves real
  Telegram traffic.
- Arbitrary malicious in-process Python is not process-isolated.
- Primary uninstall remains disabled without local activation, and purge remains
  unsupported.

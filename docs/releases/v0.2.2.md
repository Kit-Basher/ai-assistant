# Personal Agent v0.2.2 Release Notes

Status: final release audit candidate. The final `v0.2.2` tag has not been
created by the audit.

## Highlights

- Central Capability Policy Schema v1 and authorization gate.
- Universal Mutation Plan v1 for mutating operations.
- Executor Registry bindings for migrated first-party mutators.
- Package install, cleanup, update, uninstall, backup, restore, support bundle,
  memory lifecycle, bounded file mutation, Git commit, fixture-scoped service
  restart, and notification mutation authorization.
- Skill-Pack Permission Boundary v1 with declared, granted, and effective
  permissions.
- Generic Mutation Bypass Hardening v1 across shell, filesystem, Git, service,
  provider, database, secret, API, background, and fixture paths.
- Full Adversarial Authorization Proof v1: 10/10 properties proven with zero
  release blockers.
- Runtime Latency Warning Investigation and Closure v1 with evidence-backed
  accepted latency variance.
- Full pytest closure: the original `93 failed, 2406 passed` default-suite
  result was inventoried and classified; skipped-test debt closure resolved the
  89 non-environmental entries so the current default suite reports
  `2477 passed, 22 skipped, 0 failed`.

## User-Visible Changes

- Mutating actions show a structured Plan before execution.
- Confirmation is bound to the exact Plan, target, capability, executor,
  thread/session, policy, and activation state where applicable.
- Read-only inspection remains immediate where implemented.
- Status and receipts report durable mutation truth instead of relying on chat
  memory.
- Skill packs do not receive mutation permissions merely because they are
  installed; grants are scoped and default-deny.

## Operator Changes

- New readiness and audit gates cover capability policy, Universal Plans,
  generic bypasses, skill-pack permissions, adversarial authorization, runtime
  latency, default pytest closure, version consistency, upgrade compatibility,
  artifacts, and final release truth.
- Skipped-test debt gates now require `NON_ENVIRONMENTAL_DEBT=0`; remaining
  skips are environment-dependent exact node ids with replacement gates.
- `prove_ready.py` distinguishes unresolved warnings from accepted warnings and
  notes.
- Primary uninstall remains disabled unless a local operator creates a valid
  activation marker.
- Purge uninstall remains unsupported.

## Compatibility

- Product version is `0.2.2`; schema versions remain independent and are not
  renamed for the product release.
- Existing user state, preferences, memory summaries, notification history,
  Plan/receipt/grant stores, Backup v1, and Restore v1 remain compatible.
- Older skill packs without supported manifests or grants fail closed through
  the platform API permission boundary.
- Unsafe legacy behavior is intentionally denied rather than silently migrated.

## Known Limitations

- Arbitrary malicious in-process Python is not process-isolated. The guarantee
  is a permission boundary around supported platform APIs and trusted runtime
  boundaries.
- Real Git push execution is classified but disabled.
- Destructive Git variants and broad service-control operations remain denied
  unless a dedicated bounded executor and proof exist.
- Email and calendar providers are not implemented; notification migration
  covers the currently implemented communications surface.
- Runtime latency has accepted local variance for status/chat orchestration and
  cold observability. Revisit triggers are recorded in
  `docs/operator/RUNTIME_LATENCY_ACCEPTANCE_V1.json`.
